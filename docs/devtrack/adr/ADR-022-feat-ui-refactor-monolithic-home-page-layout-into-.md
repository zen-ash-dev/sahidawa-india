# ADR — feat(ui): refactor monolithic home page layout into atomic accessible…

> **Date:** 2026-06-02 | **PR:** #1127 | **Status:** Accepted

## Context

The SahiDawa frontend application's landing page suffered from a monolithic codebase structure, leading to significant technical debt, challenges in maintaining cross-device responsiveness, limitations in implementing micro-animations, and difficulties in applying consistent accessibility criteria, particularly for critical live alert dashboard cards. This architecture hindered future development, maintainability, and overall user experience.

## Decision

The monolithic homepage layout was refactored into a highly modular, component-based architecture. This involved separating distinct structural scopes into isolated client components, specifically: `Header.tsx`, `MobileNav.tsx`, `QuickActions.tsx`, and `LiveAlerts.tsx`. The implementation also included:

*   Styling global navigational segments with Tailwind backdrop blur modifiers for a glassmorphic UI.
*   Integrating colorblind-friendly accessibility features, such as distinct status textual triggers (`[🔴 CRITICAL]`, `[🟠 WARNING]`) alongside border-color variations for CDSCO real-time data panels.
*   Implementing a persistent, fluid mobile-bottom navigation shelf with enhanced touch target guidelines, optimized for small viewport screens.
*   Rewriting the entry landing component to replace bloated structural markup with a clean, import-module layout pattern.

| **Adopt a Different UI Framework/Library for Specific Sections** | While potentially offering specialized tools for certain UI elements, introducing a separate UI framework or library (e.g., React Native Web, Vue.js) for specific components within the existing Next.js/React application was rejected. This would introduce significant overhead in terms of bundle size, learning curve, tooling complexity, and potential for inconsistent styling and behavior across the application, undermining the goal of a unified and optimized frontend. |

## Consequences

**Positive:**
- Reduced technical debt and improved code maintainability through component isolation.
- Enhanced cross-device responsiveness and adaptability across various screen sizes.
- Improved user experience with native-like glassmorphic UI elements and enhanced micro-animations.
- Significantly improved accessibility for colorblind users on critical live alert dashboards.
- Optimized mobile navigation with a persistent, fluid bottom shelf and improved touch targets, enhancing PWA usability.
- Cleaner, more readable, and less bloated entry landing component file.

**Trade-offs:**
- Increased number of individual component files, requiring careful organization and naming conventions.
- Initial development time investment for refactoring the existing monolithic structure.
