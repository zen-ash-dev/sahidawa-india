# ADR — Feat/real nearby pharmacy search using supabase+post gis

> **Date:** 2026-05-29 | **PR:** #857 | **Status:** Accepted

## Context

The platform previously relied on a mock or fallback-only approach for pharmacy search, lacking real-time, efficient, and accurate geospatial capabilities. This limited the ability to provide users with nearby pharmacy locations or display pharmacies within a specific map viewport, hindering a critical feature for rural health access. The existing system did not leverage database-level geospatial indexing, leading to inefficient processing for location-based queries.

## Decision

The architectural decision was to implement real-time, efficient pharmacy search using Supabase's PostgreSQL database with the PostGIS extension. This involved:

1.  **Database-level Geospatial Functions:** Created two PostGIS Remote Procedure Call (RPC) functions:
    *   `get_nearest_pharmacies(query_lat, query_lng, search_radius_km)`: Utilizes `ST_DWithin` for indexed radius filtering and `ST_Distance` for accurate sorting.
    *   `get_pharmacies_in_bounds(bound_south, bound_west, bound_north, bound_east)`: Uses `ST_MakeEnvelope` and `ST_Intersects` for bounding box queries.
    Both functions were configured with `SECURITY DEFINER` to allow execution via the `anon` key.
2.  **Initial Data Seeding:** Seeded 30 real Pradhan Mantri Bhartiya Jan Aushadhi Kendra locations across major Indian cities with actual GPS coordinates, establishing a foundational dataset.
3.  **API Route Refactor:** The `apps/api/src/routes/pharmacies.ts` route was refactored to:
    *   Directly invoke the new PostGIS RPC functions.
    *   Pass the `search_radius_km` parameter to PostGIS for server-side filtering.
    *   Replace `any` types with strong TypeScript interfaces (`PharmacyRow`, `PharmacyRpcResult`, `FormattedPharmacy`).
    *   Add comprehensive OpenAPI/Swagger JSDoc annotations for `/api/pharmacies/nearest` and `/api/pharmacies/in-bounds`.
4.  **Graceful Fallback:** Maintained a fallback mechanism to client-side Haversine distance calculation if the PostGIS RPC functions are unavailable (e.g., extension not enabled), ensuring system resilience.
5.  **Enhanced Testing:** Expanded test coverage for the pharmacy routes, including PostGIS RPC happy paths, parameter handling, and fallback scenarios.
