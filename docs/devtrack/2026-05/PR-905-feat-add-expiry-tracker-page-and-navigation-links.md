## What Changed

We have introduced a new client-side "Medicine Expiry Tracker" page to the SahiDawa web application. This page, located at `apps/web/app/[locale]/expiry-tracker/page.tsx`, allows users to locally track the expiry dates of their medicines. To ensure easy access, a dedicated navigation link for the Expiry Tracker has been integrated into the `Footer.tsx` component.

## The Problem Being Solved

Before this PR, our platform lacked a dedicated, user-friendly tool for tracking medicine expiry dates. In a rural health context, managing medicine stock and ensuring the timely use or disposal of expired drugs is critical for patient safety and efficient resource management. Users had no built-in way to log and monitor their medicine inventory's shelf life, leading to potential waste or, more critically, the accidental use of expired medication. This feature addresses a fundamental need for local medicine management.

## Implementation Details

The core of this feature is the new `ExpiryTrackerPage` component, created at `apps/web/app/[locale]/expiry-tracker/page.tsx`. This is a client-side React component, indicated by the `"use client"` directive, leveraging Next.js's App Router for routing.

1.  **State Management and Persistence:**
    - The page uses React's `useState` hook to manage the list of `medicines` (an array of `Medicine` objects, each with `id`, `name`, `expiryDate`, and an optional `batchNumber`).
    - An `useEffect` hook handles data loading: upon component mount, it attempts to retrieve previously saved medicine data from the browser's `localStorage` using the key `'sahidawa_expiry_tracker'`. The `isLoaded` state variable tracks whether data has been loaded, allowing for a loading indicator.
    - The `saveToLocalStorage` helper function updates the `medicines` state and persists the current list to `localStorage` as a JSON string.

2.  **Medicine Management Logic:**
    - **`handleSubmit`:** This function is triggered when the "Add to Tracker" form is submitted. It prevents default form submission, validates that `name` and `expiryDate` are provided, creates a new `Medicine` object with a unique `id` (generated using `Date.now().toString()`), adds it to the `medicines` array, and then calls `saveToLocalStorage`. Form fields are reset after submission.
    - **`handleDelete`:** This function takes a medicine `id`, filters the `medicines` array to remove the matching item, and then calls `saveToLocalStorage` to update the state and `localStorage`.
    - **`getExpiryStatus`:** This utility function calculates the difference in days between a given `expiryDate` string and the current date. It returns an object containing:
        - `icon`: A `lucide-react` icon (`XCircle`, `AlertTriangle`, `CheckCircle2`) representing the status.
        - `text`: A descriptive string ("Expired", "Expiring soon", "Safe").
        - `color`: Tailwind CSS classes for styling the status badge based on urgency (red for expired, amber for expiring soon, emerald for safe).
        - Medicines expiring in less than 0 days are "Expired".
        - Medicines expiring within 30 days (inclusive) are "Expiring soon".
        - Medicines expiring in more than 30 days are "Safe".

3.  **User Interface:**
    - The page utilizes the existing `PageHeader` component, providing a consistent title, subtitle, and back navigation.
    - The main content area is structured with a responsive grid (`md:grid-cols-3`) to display an "Add Medicine" form on one side and the "Tracked Medicines" list on the other.
    - The "Add Medicine" form includes input fields for medicine `name`, `expiryDate` (using `type="date"`), and `batchNumber`.
    - The "Tracked Medicines" section displays the total count of medicines and dynamically renders each medicine item. It shows a loading message (`animate-pulse`) or an empty state message with a `Package` icon if no medicines are tracked.
    - Each tracked medicine item displays its name, formatted expiry date, and a status badge generated by `getExpiryStatus`. A `Trash2` icon button allows users to delete individual entries.

4.  **Navigation Integration:**
    - In `apps/web/app/[locale]/components/Footer.tsx`, we imported the `CalendarRange` icon from `lucide-react`.
    - A new `Link` component from `@/i18n/routing` was added to the "Quick Links" section of the footer. This link has an `href` of `/expiry-tracker` and displays "Medicine Expiry Tracker" alongside the `CalendarRange` icon, styled with `text-emerald-400`.

## Technical Decisions

1.  **Client-Side Persistence with `localStorage`:** We opted for `localStorage` for data storage. This decision was made to keep the feature entirely client-side, aligning with its "beginner" level and allowing for rapid deployment without requiring backend API changes or database integration. This makes the feature self-contained and immediately functional for individual users.
2.  **Next.js App Router and `"use client"`:** The page is built within the Next.js App Router structure (`app/[locale]/expiry-tracker/page.tsx`), which is our standard for new pages. The `"use client"` directive is necessary because the page relies heavily on React hooks (`useState`, `useEffect`) and browser-specific APIs (`localStorage`), which must execute in the client environment.
3.  **`lucide-react` for Icons:** We consistently use `lucide-react` for vector icons across the SahiDawa frontend. This PR continues that pattern by importing `CalendarRange`, `Trash2`, `Package`, `XCircle`, `AlertTriangle`, and `CheckCircle2` for visual cues on the Expiry Tracker page and in the footer navigation.
4.  **Tailwind CSS for Styling:** All styling is implemented using Tailwind CSS utility classes. This maintains consistency with the existing SahiDawa design system, ensuring the new page integrates seamlessly with the application's look and feel, including dark mode support.
5.  **Simple ID Generation:** For local data management, `Date.now().toString()` is used to generate unique IDs for medicine entries. While sufficient for client-side `localStorage`, a more robust UUID generation strategy or backend-assigned IDs would be used if this feature were to integrate with a database.
6.  **Comprehensive Navigation:** Placing the link in the `Footer.tsx` ensures that the Expiry Tracker is easily discoverable and accessible from any page within the application, enhancing user experience.

        ```typescript
        interface Medicine {
            id: string;
            name: string;
            expiryDate: string;
        ```

    - Implement a `useEffect` hook to load data from `localStorage` on component mount, parsing the JSON string stored under the key `'sahidawa_expiry_tracker'`. Set `isLoaded` to `true` after the load attempt.
    - Create the `saveToLocalStorage` function that accepts an `updatedList: Medicine[]`, updates the `medicines` state, and then calls `localStorage.setItem('sahidawa_expiry_tracker', JSON.stringify(updatedList))`.

        - Prevent default form submission (`e.preventDefault()`).

    - Implement `handleDelete(id: string)` to remove medicines:
        - Filter the `medicines` array to exclude the item with the given `id`.
        - Call `saveToLocalStorage` with the filtered array.
    - Implement `getExpiryStatus(dateStr: string)`:
        - Parse `dateStr` into a `Date` object.
        - Calculate the difference in days from the current date.
        - Return an object with `icon` (from `lucide-react`), `text` (e.g., "Expired"), and `color` (Tailwind classes) based on the expiry logic (expired, expiring soon, safe).
    - Construct the JSX for the page, including the `PageHeader` component, the "Add Medicine" form (with inputs for name, expiry date, batch number, and a submit button), and the "Tracked Medicines" list.
    - Use conditional rendering for loading and empty states in the tracked medicines list.
    - Map over the `medicines` array to display each item, applying `getExpiryStatus` for visual feedback and including a delete button.
    - Ensure all necessary `lucide-react` icons (`Calendar`, `Trash2`, `Package`, `XCircle`, `AlertTriangle`, `CheckCircle2`) are imported.

## Impact on System Architecture

This change introduces a new, self-contained client-side utility feature within the `apps/web` Next.js frontend. It demonstrates a pattern for adding new interactive pages that leverage browser-specific APIs (`localStorage`) for local data persistence, suitable for features that do not require server-side storage or complex data relationships. This expands the SahiDawa platform's utility by providing a practical tool for individual users to manage medicine expiry dates, directly supporting our mission in rural health. While currently isolated, this feature lays foundational UI/UX for future enhancements that might integrate with a backend for multi-user support, synchronization, or more advanced inventory management systems. It also reinforces the use of `lucide-react` for icons and Tailwind CSS for styling consistency across the application.
